## When to use this skill
To write in internal documents like pro-211 or other, use this skill for:
- Requirements
- Project risks
- Deliverables list
- INO presentation PPTX outputs
- Luqia presentation PPTX outputs

## How to use this skill

To write in internal documents:

1. **Identify the communication type** from the request, the document name or a quick glance at its content
2. **Load the appropriate guideline file** from the `examples/` directory:
    - `examples/requirements.md` - For Requirements, stakeholder needs, and front-facing Excel reference hygiene.
    - `examples/project-risks.md` - For Project risks.
    - `examples/improvement-list.md` - For improvement list over the course of a developement.
    - `examples/presentation-pptx.md` - For INO or Luqia PPTX presentation outputs.
    - `examples/ITR-399-755.docx` - Default template for unspecified or underdefined INO or Luqia internal documents.
3. **Follow the specific instructions** in that file for formatting, tone, and content gathering

If the request is for an INO or Luqia document but the format is unspecified or underdefined, use `examples/ITR-399-755.docx` as the starting template by default, then infer the missing structure from the request and preserve the template's style.

If the communication type doesn't match any existing guideline and is not an unspecified or underdefined INO or Luqia document, ask for clarification or more context about the desired format.

French is a common language used, ensure encoding, extraction and editing support accentuated letters.
