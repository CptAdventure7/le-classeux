# 09 Communication

Project communication artifacts live here.

## Folders

- `meetings`: working session records, raw notes, and cleaned summaries
- `internal_updates`: updates intended for internal project stakeholders
- `external_updates`: updates intended for customers, partners, or other outside audiences
- `reporting`: recurring status reports, summaries, or KPI-style reporting artifacts

## Authoring Rules

- Put communication artifacts in the narrowest matching child folder.
- Keep the message itself here; keep source decisions, requirements, and other canonical records in their own sections and link to them.
- Prefer updating an existing report, meeting thread, or update thread over creating near-duplicate summaries.
