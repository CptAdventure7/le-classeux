# Internal Updates

Internal-facing project updates live here.

## Keep Here

- team updates
- leadership or steering updates meant for internal readers
- internal status notes that summarize recent work, risks, or next steps

## Authoring Rules

- Keep audience-facing communication here, not raw meeting notes or decision records.
- Link to the requirement, issue, meeting summary, or decision that supports the update instead of repeating all of that context.
- If the same update is revised over time, update the existing artifact when practical.

## Local Manifest

- Review `local_manifest.yaml` when entering this folder to quickly see whether any local files are relevant.
- Keep `local_manifest.yaml` up to date whenever files are added, removed, renamed, or materially repurposed.
