# Governance

This folder contains the governance entrypoints for the active project template.

## Keep Here

- `change_log/` for append-only project-level traceability
- `current_overview/` for compact current-state snapshots and routing
- `current_overview/runtime_scan_summary.json` for the cached metadata-only scan that helps agents avoid spending tokens on scaffold-only sections
- this `README.md`

## Keep Elsewhere

- project evidence that belongs in a narrower active section
- duplicate summaries that should instead update `current_overview/` or the project root `README.md`
- history-only material that belongs in `99_ARCHIVE`, which should stay outside the normal runtime scan baseline unless the query is explicitly about project history
