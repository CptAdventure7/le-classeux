# Experiments

One markdown file per experiment.

- If new material belongs to an existing experiment, update that file instead of creating a new one.
- Use a meaningful filename that reflects the experiment.
- Link the experiment to the related protocol, result, or analysis when relevant.

## Local Manifest

- Review `local_manifest.yaml` when entering this folder to quickly see whether any local files are relevant.
- Keep `local_manifest.yaml` up to date whenever files are added, removed, renamed, or materially repurposed.
