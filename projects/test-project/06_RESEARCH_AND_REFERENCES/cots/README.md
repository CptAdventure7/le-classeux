# Cots

Commercial off-the-shelf component comparison artifacts live here.

## Keep Here

- Create one subfolder per component area when comparing options for the same subsystem or part family.
- In each component-area folder, keep the structured comparison artifact for that area plus the notes and source evidence needed to justify the shortlist.
- Add a deeper subfolder per specific candidate or vendor only when downloaded reference material, quotes, or raw source captures need to stay grouped together.
- Keep links to affected requirements, design records, procurement items, decisions, or validation evidence close to the comparison instead of scattering them across folders.

## Keep Elsewhere

- Put general web research that is not tied to a specific COTS comparison in `../web_references`.
- Put procurement execution artifacts such as quote approval flow or purchase timing in `../../07_PROJECT_EXECUTION/procurement`.
- If design-owned mirrored vendor material is maintained, keep it in the matching area under `../../04_DESIGN_AND_IMPLEMENTATION/design`.

## Example

`motor-controllers/` containing the structured options comparison for that component area, normalized notes, and optional candidate folders such as `vendor-a-model-x/` for datasheets or quoted material.

## Local Manifest

- Review `local_manifest.yaml` when entering this folder to quickly see whether any local files are relevant.
- Keep `local_manifest.yaml` up to date whenever files are added, removed, renamed, or materially repurposed.
