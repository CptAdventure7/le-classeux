# External Updates

External-facing project updates live here.

## Keep Here

- customer or partner updates
- outbound release or progress summaries
- communication prepared for readers outside the core project team

## Authoring Rules

- Keep only the outward-facing communication here; store internal working notes elsewhere.
- Link to the supporting source material instead of copying large blocks of background context.
- If a message is tailored from an internal update, keep one canonical external version here rather than duplicating drafts across folders.

## Local Manifest

- Review `local_manifest.yaml` when entering this folder to quickly see whether any local files are relevant.
- Keep `local_manifest.yaml` up to date whenever files are added, removed, renamed, or materially repurposed.
