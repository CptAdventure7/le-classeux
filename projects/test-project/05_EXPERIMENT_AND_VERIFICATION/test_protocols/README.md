# Test Protocols

One markdown file per test protocol.

- If new material belongs to an existing protocol, update that file instead of creating a new one.
- Use a meaningful filename that reflects the protocol.
- Link the protocol to the related experiment or result when relevant.

## Local Manifest

- Review `local_manifest.yaml` when entering this folder to quickly see whether any local files are relevant.
- Keep `local_manifest.yaml` up to date whenever files are added, removed, renamed, or materially repurposed.
