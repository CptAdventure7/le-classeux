# User Docs

Keep user-facing guides, quick starts, workflows, and reference material here.

- Write for the end user, not the implementation team.
- Link to training material or deployment and installation steps when the user needs them.

## Local Manifest

- Review `local_manifest.yaml` when entering this folder to quickly see whether any local files are relevant.
- Keep `local_manifest.yaml` up to date whenever files are added, removed, renamed, or materially repurposed.
